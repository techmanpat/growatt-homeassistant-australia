Phase 1, Installing Docker and Running Home Assistant (WSL2)

This guide installs WSL2, Docker Desktop, and runs Home Assistant with persistent storage on your local drive. It is written for Windows 11, using PowerShell and Docker Desktop with WSL2 integration.

Target outcome

Home Assistant runs in Docker, and survives reboots and container re creation.

Timezone

Australia/Perth

Non negotiable

/config must be mapped to C:\HomeAssistant\config

Contents

  • Part A, Install WSL2 properly
  • Part B, Install Docker Desktop
  • Part C, Create a permanent Home Assistant folder
  • Part D, Run Home Assistant with persistent storage
  • Part E, Backups and recovery
  • Part F, Prevent “Docker WSL storage resets”
  • The non negotiable rule

Tip, open this page on one screen, and follow the steps on your Windows machine on the other.

Part A, Install WSL2 properly

Step 1, Enable WSL2

  1. Open PowerShell as Administrator.
  2. Run the command below.
  3. Reboot when prompted.
PowerShell (Admin) 
wsl --install

Optional, Step 2, Confirm WSL2 is running

  1. After reboot, open PowerShell (normal is fine).
  2. Run the command below to list distros and their version.
  3. You should see a distro, usually Ubuntu, and Version 2.
PowerShell 
wsl -l -v

If your distro shows Version 1, run these commands.

PowerShell 
wsl --set-default-version 2
wsl --set-version Ubuntu 2

Part B, Install Docker Desktop

Step 3, Install Docker Desktop

  1. Download and install Docker Desktop using the official installer.
  2. During install, select “Use WSL 2 instead of Hyper-V” if prompted.
  3. After install, open Docker Desktop once.

Step 4, Enable WSL integration

  1. Open Docker Desktop.
  2. Go to Settings, then Resources, then WSL Integration.
  3. Enable integration for your distro, for example Ubuntu.
  4. Click Apply and Restart.

Step 5, Set Docker disk location on your local drive (optional but smart)

  1. Docker Desktop, Settings, Resources, Advanced.
  2. Increase Disk image size, 200GB+ recommended.
  3. If Docker offers “Disk image location”, set it to a folder on C: or D: that you control.
  4. Click Apply and Restart.

Even if Docker’s internal disk lives elsewhere, your Home Assistant config is safe as long as you bind mount /config in Part D.

Part C, Create a permanent Home Assistant config folder on C:

Step 6, Create folders

  1. Open PowerShell (normal).
  2. Create the folders below.
PowerShell 
mkdir C:\HomeAssistant -Force
mkdir C:\HomeAssistant\config -Force
mkdir C:\HomeAssistant\backups -Force
What this does
This is where your entire Home Assistant setup lives, config, storage, logs, and backups, independent from Docker internals.

Part D, Run Home Assistant in Docker with persistent storage

Step 7, Run Home Assistant with bind mount (persistent)

  1. Open PowerShell.
  2. Run this docker command to start Home Assistant.
PowerShell 
docker run -d `
  --name homeassistant `
  --restart unless-stopped `
  --privileged `
  -e TZ="Australia/Perth" `
  -p 8123:8123 `
  --log-opt max-size=10m `
  --log-opt max-file=3 `
  -v C:\HomeAssistant\config:/config `
  homeassistant/home-assistant:stable

Step 8, Confirm files are being written to C:

  1. Wait 1 to 2 minutes.
  2. Run the command below.
  3. You should see Home Assistant files appear in that folder.
PowerShell 
dir C:\HomeAssistant\config

You should see items like configuration.yaml, a .storage folder, and home-assistant.log. That means persistence is working.

Step 9, Test persistence (proves you won’t lose config)

  1. Stop and remove the container.
  2. Then re run the same docker run command from Step 7.
  3. Confirm Home Assistant opens in the same instance, not onboarding.
PowerShell 
docker stop homeassistant
docker rm homeassistant

Then open: http://localhost:8123

Part E, Make backups so you can recover from anything

Step 10, Create Home Assistant backups

  1. In Home Assistant, go to Settings, System, Backups.
  2. Create a backup.
  3. Download the backup file periodically.
  4. Save it to: C:\HomeAssistant\backups

If you want automation later, you can schedule copy jobs or sync backups to cloud storage.

Part F, Things that prevent “weird Docker WSL storage resets”

Step 11, Keep Docker logs from growing forever

This is already handled by the docker run flags: --log-opt max-size=10m, and --log-opt max-file=3.

Step 12, Periodic cleanup (optional)

  1. Once a month, check disk usage.
  2. Optionally prune unused docker artifacts.
PowerShell 
docker system df
docker system prune -a
Be careful
prune -a removes unused images. Only use it if you understand what it will delete.

The non negotiable rule

If you only remember one thing
If /config is not mapped to C:\HomeAssistant\config, you can lose everything.
With the bind mount, restarts and reboots cannot wipe your Home Assistant config unless you delete that folder.

If you want everything on C: only, or you want Docker’s disk image moved to D: for more space, update Docker Desktop settings in Part B Step 5.
